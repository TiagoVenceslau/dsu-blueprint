# constant DSUBlueprint

DSU Blueprint Decorator

Defines a class as a DSU Blueprint, enabling:

  • Automatic CRUD operations, just by updating its corresponding DSUModel instance and running it through the appropriate OpenDSURepository;
  • Automatic validations && easily extendable for added validations;
  • Automatic serialization -> transmission -> deserialization;
  • Controlled accesses: Ability easily to add business logic at key points of any CRUD operations

# fromWeb(appOrUrl, slot, triggerMountopt, deriveopt, mountPathopt, mountOptionsopt, keyOverrideopt, argsopt)

Depending on whether and url or an app name is supplied:

  • App Name: Communicates with ApiHub to get seeds from SSApps installed in that ApiHub instance:
    • slot 'primary': gets the seed file from the App's 'wallet-patch' folder;
    • slot 'secondary' gets the seed file from the App's ConstantsApi#APP_FOLDER

Retrieves the Seed during the DSUOperation.CREATION operation and DBOperations.CREATE phases and stores in the in object under the matching property key

calls mount on this property to mount the seed during the DSUOperation.EDITING operation;

Name Type Attributes Description

the slot where the seed can be found. 'primary' refers to the 'wallet slot' and secondary to the 'apps slot'. defaults to 'primary' and is mandatory when an App Name is provided. is discarded if a URL is detected in the appName
